# 애터미 마케팅플랜 - 바이너리 규칙 및 후원수당 테이블

## 1. 바이너리 구조 기본 규칙

1. 나를 기준으로 좌/우(바이너리 형식)로 회원을 가입시킬 수 있다.
2. 하위에 내려갈 때 기본적으로 좌에 회원이 가입된다.
3. 후원수당 테이블에 의해 **매일 정산**하여 점수를 받는다.
4. 점수 × N가 = 후원수당
5. **점수를 받으면 다음날 나의 좌/우 PV는 0으로 초기화** (소실적)
6. 내 하위에서 올라온 점수는 무한단계로 누적된다.
7. 내 하위 좌그룹의 모든 회원 PV 합 = 나의 좌 PV
8. 내 하위 우그룹의 모든 회원 PV 합 = 나의 우 PV
9. 이 원리는 모든 회원 각각 본인 기준 좌/우에 동일 적용

## 2. 후원수당 테이블

| 회원PV (몸PV, 무한누적 소실적) | 좌 PV | 우 PV | 점수 | N가 | 일일 후원수당 |
|---|---|---|---|---|---|
| 30만 PV | 30만 PV | 30만 PV | 15점 | 3,500~4,000원 | 약 6만원 |
| 70만 PV | 70만 PV | 70만 PV | 30점 | | 약 12만원 |
| 150만 PV | 150만 PV | 150만 PV | 60점 | | 약 24만원 |
| 240만 PV | 240만 PV | 240만 PV | 90점 | | 약 36만원 |
| 240만 PV | 600만 PV | 600만 PV | 150점 | | 약 60만원 |
| 240만 PV | 2000만 PV | 2000만 PV | 250점 | | 약 100만원 |
| 240만 PV | 5000만 PV | 5000만 PV | 300점 | | 약 120만원 |

**계산식:**
```
일일 총 PV 합 → 일일 총 점수 합 × 44% ÷ N가 = 후원수당
```

**회원 PV 조건:** 무한누적 소실적 기준
**초기화:** 점수를 받은 다음날 좌/우 PV 초기화

## 3. 소실적 매칭 원리

- 좌/우 PV가 동시에 특정 구간을 충족할 때 점수 발생
- 예: 좌 30만 + 우 30만 → 15점 발생, 다음날 좌/우 0으로 초기화
- **비효율 예시:** 하루에 좌 250만 PV 몰아넣기 → 그날 90점만 받고 나머지는 소멸 위험
- **효율 예시:** 여러 날에 걸쳐 좌/우 30만씩 매칭 → 8번 × 15점 = 120점

## 4. 이틀 매칭 규칙 (유연한 배분)

하루 안에서 좌/우를 같은 날 매칭하지 않아도 된다:
- 1일차: 좌에 30만 PV 입력
- 2일차: 우에 30만 PV 입력 → 이때 매칭 발생, 점수 획득 후 초기화

**결론:** 하루 매칭도 가능, 이틀 매칭도 가능 → 유연하게 운영

## 5. 롤업(Roll-up) 원리

- 하위 직급자의 PV는 상위 직급자의 좌/우로 그대로 합산됨
- 하위 SM이 특정 날짜에 배치한 몸PV는 상위 DM의 그날 좌/우 PV에도 반영됨
- 이를 활용하면 상위 DM의 일일 수당 점수를 15점→30점으로 올릴 수 있음

## 6. 플래시아웃(Flash-out) 방지

- 좌 또는 우 한쪽만 계속 쌓이고 매칭이 없으면, 초기화 시 소멸
- 직급표 설계 시 좌/우 균형 유지 필수
- 몸PV 배치는 플래시아웃 위험이 없는 날짜에 배치할 것
